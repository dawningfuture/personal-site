Angular Frontend
I built this site using Angular open_in_new, which I have been using to build projects since 2017. Everyone has their favorite tools; I chose Angular for this project because of its modularity (see the libraries I chose to use), its features (such as the lazy-loading capability of the Router library), and the familarity I have gained with it after 4+ years of using it.
I did my best to show the Angular best practices that I have learned over the years on this site. The code is publicly available on GitHub open_in_new, and I did my best as I was developing the site to document my thoughts on the best practices and methodoligies I was using in code comments.
I used several libraries from both the Angular team and the community that let me achieve many of the site's features, such as:
@ngrx/[store | effects | router-store | store-devtools] open_in_new
When I discovered NgRx, I fell in love with the library's reactive store approach to state management. One of the most impactful lessons I have learned from working on frontend projects is the need for a consistent, cohesive, and modular way to communciate between services and components, and maintain a centralized state for the application. You can see it in action using the Redux DevTools Chrome extenstion open_in_new.
@angular/material open_in_new
This library gives me the basic styles of this site, which I override to incorporate my own design. When I'm developing, I try not to reinvent the wheel. Angular Material provides a styling system that sets the baseline for the look of the site's components, but gives me the freedom to make the site my own.
@angular/router open_in_new
One feature I wanted the site to have was lazy-loaded pages. The Angular Router library includes this out-of-the-box. It's an example of why I so enjoy building projects with Angular: while initially there may be a lot learn, eventually you learn how much there is to benefit from.
@angular/animations open_in_new
I want the projects I work on to bring joy to people, and I find that is helped greatly when I can create an appealing experience. I use this library to more easily add the animations that make the site more enjoyable to interact with.
@angular/forms open_in_new
I know some people despise building forms, but I've come to love it. This library makes common, yet difficult, form tasks like validation and error messaging easy. I use it to create the form on the Connect page; feel free to play with it to see what I'm talking about!
@angular/flex-layout open_in_new
I really enjoy using Flexbox CSS to design my layouts; this library allows me to integrate a responsive Flexbox API into Angular to make my site optimized for whatever screen it's being viewed on. I especially enjoy its breakpoint system, which makes mobile optimization much easier.
